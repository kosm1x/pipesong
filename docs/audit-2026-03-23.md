## Critical Finding: Kokoro TTS Latency Root Cause

**The 800-1600ms in-pipeline TTFB vs 115ms standalone is caused by Pipecat's default `SENTENCE` text aggregation mode.** It buffers ALL LLM tokens until a sentence boundary (`.` `?` `!`) before sending to TTS. Kokoro sits idle while the LLM generates an entire sentence.

**Fix:** Set `text_aggregation_mode` on the TTS service:
```python
from pipecat.services.tts_service import TextAggregationMode  # verify import path

tts = KokoroTTSService(
    voice_id=voice,
    text_aggregation_mode=TextAggregationMode.TOKEN,  # or clause-level
    settings=KokoroTTSService.Settings(voice=voice, language="es"),
)
```

**Caveat:** TOKEN mode may produce choppy audio with Kokoro (local model, not optimized for single-token synthesis). Test clause-level aggregation (release on commas, conjunctions) as a middle ground.

**Expected impact:** 500-1000ms latency reduction. Should bring total pipeline to ~1s.
